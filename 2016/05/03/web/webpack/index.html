webpack 使用

May 3, 2016 | web | Hits

使用webpack

默认情况

打包所有文件到js,适合文件较少时

大项目

分离打包JS和CSS文件，如有需要可以创建code splitting，用来做异步加载
如果不是单页应用，需要做多个入口文件，可以使用commonchunk插件做公共文件提取

loaders

  1. file-loader 加载图片资源
  2. babel 用来使用es6（主意code splitting）
  3. style-loader，css-loader，stylus-loader 解析stylus,css文件
  4. vue-loader 使用vuejs时使用
  5. bundle-loader code splitting loader
  6. json-loader 加载JSON文件
  7. raw-loader 加载普通文件
  8. write loader

plugins

  1. CommonsChunkPlugin 用来做公共资源提取
  2. ExtractTextPlugin 用来做css分离
  3. UglifyJsPlugin 混淆压缩js
  4. OpenBrowserPlugin 自动打开浏览器
  5. HtmlwebpackPlugin自动生成index.html
  6. DefinePlugin 定义全局变量，用来判断环境等等
  7. ProvidePlugin 创建在所有模块中可以使用的对象
  8. write plugin

webpack-cli

  1. –display-modules 显示隐藏的模块，用来分析问题
  2. –progress 显示构建过程
  3. –display-error-details 显示错误信息
  4. –profile 输出性能数据，可以看到每一步的耗时
  5. –colors 输出结果带彩色

config

  1. publishPath 配置发布根目录，可以使用cdn地址
  2. chunkFilename 用来配置不在entry文件里面的文件名称，主要用来和code splitting配合
  3. resolve 使用模块配置优化执行，常用alias,extensions
  4. module.noParse 用来优化不需要解析的库

other

  • require.ensure 用来做异步加载实现code splitting
  • loader加载顺序为从右向左(style-loader!css-loader 先加载css,在通过管道传入style)
  • resolveLoader 如果需要自定义loader目录，请修改resolveLoader.root
